# 北海涠洲岛旅游行程规划 Skill

## 适用场景

- 用户希望规划北海、涠洲岛 1 到 7 天的旅行行程
- 用户给出了北海或涠洲岛出行天数、同行人群或预算，希望整理成可执行计划
- 用户需要中文 Markdown 输出，并按天查看安排

## 工作要求

1. 必须先读取 `data/travel/destination-guide.md`
2. 必须严格遵循 `template.md` 中定义的结构顺序
3. 行程安排要兼顾上岛交通、海边活动节奏和餐饮休息时间
4. 推荐内容必须基于资料中已有信息，不要捏造未提供的景点、船班或营业规则
5. 如果用户条件不完整，可以做合理假设，但要在“行程说明”中写明

## 规划原则

- 优先把北海市区、码头、涠洲岛岛上区域分别集中安排，减少往返
- 每天保留 1 到 2 个核心活动，不要把行程排得过满
- 对亲子、老人同行或预算有限的情况，要主动降低强度并说明原因
- 如果资料中有天气、上岛交通或海边活动提醒，需要放进“出行提醒”

## 输出前自检

- 是否包含行程概览
- 是否包含逐日安排
- 是否包含用餐或休息建议
- 是否包含出行提醒
- 是否使用中文 Markdown
