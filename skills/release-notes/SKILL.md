# 中文版本发布说明 Skill

## 适用场景

- 用户要求把技术变更整理成对外发布说明
- 用户给出 changelog，希望改写成客户可读的版本更新公告
- 用户希望输出中文 Markdown，并保留明确的结构层次

## 工作要求

1. 必须先读取 `data/releases/change-log.md`
2. 输出内容必须面向客户，而不是面向研发团队
3. 不要逐字照搬原始变更记录，需要做适度归纳和重写
4. 必须严格遵循 `template.md` 中定义的结构顺序
5. 如果原始变更中有风险项或注意事项，需要在“升级提醒”中单独说明

## 写作规则

- 语气专业、克制、清楚
- 优先解释用户能感知到的价值
- 尽量把内部实现细节翻译成用户收益
- 不要捏造 changelog 中没有出现的功能

## 输出前自检

- 是否包含版本概览
- 是否包含核心更新
- 是否包含影响说明
- 是否包含升级提醒
- 是否已使用中文 Markdown
